import { Code } from 'astro/components';

I'll set up a Next App with Typescript using the `create-next-app` command

<Code
  code={`npx create-next-app@latest --ts`}
  theme="dark-plus"
  lang="zsh"
  wrap
></Code>
Move pages & styles to source folder
This is how I set up my tsconfig.json, the paths options let's me use `@components/componentName` 
on imports instead of `../../components/componentName`
<Code
code={`{
  "compilerOptions": {
    "target": "es5",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "skipLibCheck": true,
    "strict": true,
    "forceConsistentCasingInFileNames": true,
    "noEmit": true,
    "esModuleInterop": true,
    "module": "esnext",
    "moduleResolution": "node",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "jsx": "preserve",
    "incremental": true,
    "baseUrl": ".",
    "paths": {
      "@components/*": ["src/components/*"],
      "@utils/*": ["src/utils/*"]
    }
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx"],
  "exclude": ["node_modules"]
}`}
  theme="dark-plus"
  lang="json"
  wrap
 />

<Code
  code={`cd <project-name>`}
  theme="dark-plus"
  lang="zsh"
  wrap
></Code>
Next, I'll install [Tailwind](https://tailwindcss.com) for this project,
Tailwind is a CSS framework that allows for rapid protyping without adding any bloated (mostly unused) CSS

<Code
  code={`npm install -D tailwindcss postcss autoprefixer`}
  theme="dark-plus"
  lang="zsh"
  wrap
></Code>
Then I'll create a Tailwind config file.
<Code
  code={`npx tailwindcss init -p`}
  theme="dark-plus"
  lang="zsh"
  wrap
></Code>

Now I'll install [ESLint](https://eslint.org/) a tool used to catch common JS/TS bugs.
<Code code={`npm run lint`} theme="dark-plus" lang="zsh" wrap></Code>

Create a eslintrc.json file. This is a config file where all ESLint related settings live.
<Code
code={`{
  "extends": ["next/core-web-vitals", "prettier"]
}`}
  theme="dark-plus"
  lang="json"
  wrap
 />

Create .eslintignore file, this works similar to a .gitignore & basically tells eslint
to ignore these files/directories while linting

 <Code
code={`# dependencies
/node_modules
/.pnp
.pnp.js

# testing
/coverage

# next.js
/.next/
/out/

# production
/build
`}
  theme="dark-plus"
  lang="zsh"
  wrap
 />

Lastly I'll install [Prettier](https://prettier.io/), a tool used to catch automatically format my code.
<Code
  code={`npm install -D eslint-config-prettier`}
  theme="dark-plus"
  lang="zsh"
  wrap
></Code>

Create a prettierrc.json file. This is a config file where all Prettier related settings live, 
similar to what I did with the eslintrc.json file .
<Code
code={`{
  "useTabs": false,
  "tabWidth": 2,
  "singleQuote": true,
  "printWidth": 60,
  "trailingComma": "es5"
}
`}
  theme="dark-plus"
  lang="json"
  wrap
 />

Create .prettierignore file, this works similar to a .gitignore or the .prettierignore file mentioned earlier.
It basically tells prettier to ignore these files/directories when formatting code.
 <Code
code={`# dependencies
/node_modules
/.pnp
.pnp.js

# testing
/coverage

# next.js
/.next/
/out/

# production
/build
`}
  theme="dark-plus"
  lang="zsh"
  wrap
 />
